Initializing Git    
git init

Adding files to staging area (preparing changes to be saved permanently): 
git add . or git add fileName

Command to check the status of your project files: git status

git commit -m "added commands to git commands text file and created an html file (notes)" 

Commit: A snapshot of your project at a moment in time
git commit -m "message"

use git commit format in order to share keywords and messages for detailed update notes
look online for keywords and best practices for format
you can use emojis to also represent the changes 

Commit frequently in order to track issues and progress so you can go back 

git push pushes all of the commits to the online repository

Connecting my local project in VSCode to my Github Repository in the cloud:
git remote add origin link_to_Github_repo

git push origin master (or main) whichever is labeled in the bottom left
